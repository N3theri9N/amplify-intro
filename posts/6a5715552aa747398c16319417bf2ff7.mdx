## BOM

**Browser Object Model | 브라우저 객체 모델**

웹 브라우저를 자바스크립트에서 제어하기 위한 객체이다. 객체 종류는 다음과 같다

- window : 최상위 객체, **모든 객체가 소속된 객체**이다. 아래의 객체를 모두 포함한다.
    - document : 현재 문서에 대한 정보
    - history : 현재의 브라우저가 접근하던 URL 이력을 제어
    - location : 문서의 주소를 다루는 객체, window 객체와 document 객체의 prop이다.
    - screen : 사용자 디스플레이 화면에 대한 정보
    - navigator : 실행중인 앱에 대한 정보이며 크로스 브라우징 이슈를 해결할 때 사용

## Load, Defer, Async

자바스크립트 파일을 효과적으로 가져오는 방법을 다룬다.

### `<script>`

![](https://file.notion.so/f/f/b67b8caf-0e30-4e9a-b6db-84175dd949a3/ba154e8f-3827-4720-bb74-a8f36a12bb85/Untitled.png?id=e11d677d-2734-4a78-b67a-73b2310aa557&table=block&spaceId=b67b8caf-0e30-4e9a-b6db-84175dd949a3&expirationTimestamp=1706169600000&signature=FpgLe3Obso6teGdRNoao89tMMTYfwaC8t8t8z4qgfBI&downloadName=Untitled.png)

Html 에서는 `<script>` 태그를 통해 자바스크립트 파일을 불러들인다. 브라우저에서는 index.html 파일을 보게되면 **위에서부터 아래로 읽어들인다. `<script>`** 태그를 만나면 하던 작업을 멈추고 스크립트 리소스를 가져오고 **스크립트를 실행까지 한다**.

querySelector 로 찾을 태그가 parsing 되기 전에 스크립트를 부를 경우 분명 index.html 에서는 요소가 있지만 스크립트에서는 찾지 못하는 일이 발생한다. 

그러므로 `<script>` 태그는 주로 `</body>` 종료태그 바로 위에 넣는다. 

### window.onload

Html 파싱, DOM 그리고 외부 컨텐츠( CSS, script, img 등등 )가 로드된 후 발생하는 이벤트이다. 그러므로 모든 컴포넌트가 있을 때 참조할 수 있다. 

이 이벤트 리스너에 선언한 건 중간에 선언된 Script 일지라도 즉시 실행되지 않고 나중에 실행된다. 

### DOMContentLoaded

DOM 생성후 즉시 발생하는 이벤트이다. `document.addEventListener('DOMContentLoaded', function(){})` 으로 선언한다.

### Defer prop

`<script>` 태그에 defer 속성을 넣으면 **HTML 파싱이 완료된 후** JavaScript 코드를 실행한다.

```jsx
<script src='script.js' defer></script>
```

### Async prop

이 속성을 넣으면 비동기로 javaScript 파일을 불러들인다. 즉 이 스크립트를 실행하는 중엔 HTML 파싱이 중단된다. 그런데 이는 제어가 어려워서 권장하지 않는다.
